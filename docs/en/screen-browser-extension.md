# Browser Extension Screen

The topbar `?` opens this page from the Browser extension screen. Use it for pairing status, approved devices, extension health and Chrome Web Store update expectations.

## Work Here

- Approve only known browser profiles.
- Revoke lost, stale or unexpected devices.
- Use Chrome Web Store as the normal install/update path; ZIP files are archive, development or emergency fallback only.

## Screen States

| State | Operator response |
| --- | --- |
| Pending | Confirm origin, account and browser profile before approving. |
| Approved | Check last sync and revoke unused devices. |
| Failed | Use the extension pairing KB before reinstalling or publishing evidence. |

## Before You Act

- Match the pairing code, user request, device name and browser before approval.
- Check whether the device is managed, stale or revoked before troubleshooting sync.
- Keep extension IDs and version information public-safe; keep pairing material private.

## Safe Evidence

- Safe to share: extension ID, extension version, browser family, redacted device state and Chrome Web Store status.
- Keep private: pairing codes, device keys, host-specific screenshots, vault counts and autofill-visible record values.
- Use the Chrome Web Store listing as the normal install/update reference; ZIP packages stay fallback or lab evidence only.

## Operator Notes

Never publish pairing codes, extension device IDs, host-specific screenshots or vault record values shown through autofill.
